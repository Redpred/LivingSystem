# LivingSystem 游戏内测试清单

> 覆盖已实现的阶段二~六功能。每项含「前置 / 操作 / 预期」。建议用开发客户端
> （`./gradlew runClient`）单人世界逐项验证。许多系统为渐进效果，必要时用配置或命令加速。

## 0. 测试准备

| 项 | 说明 |
| --- | --- |
| 启动客户端 | `./gradlew runClient`（Windows 编译建议先设 `JAVA_TOOL_OPTIONS=-Duser.language=en -Duser.country=US`） |
| 调试聊天 | 服务端配置 `debugChat=true`（默认开）：命中/失血/骨折/死亡等输出到聊天框 |
| 调试命令 | `debugCommands=true`（默认开）：`/livingsystem` 无需 OP 可用 |
| 可用命令 | `/livingsystem debug`（查看血量/伤势）、`heal`（重置健康）、`creative`/`survival`（切换模式）；其余子命令为占位提示 |
| 加速观察 | 调小 `tickInterval`（默认 5），或调大相关倍率（疼痛/骨折/体温变化率）以更快看到效果 |
| 打开健康界面 | 默认按键 **H**（设置→按键绑定→生命系统 可改键） |

> 配置文件位置：`run/config/livingsystem-server.toml`（服务端规则）、`livingsystem-client.toml`（客户端表现）。

## 1. 资源桥接与 HUD（阶段二）

| # | 操作 | 预期 |
| --- | --- | --- |
| 1.1 | 进入生存模式观察 HUD | 屏幕下方文字显示「血液 X/20  体力 X/20  水分 X/20  呼吸 X/20」 |

## 2. 分部位创伤与出血（阶段二 2.1/2.2）

| # | 操作 | 预期 |
| --- | --- | --- |
| 2.1 | 让生物攻击你 / 摔落 / 中箭 | 聊天显示「命中 <部位>/<创伤类型> 严重度X 失血XmL → 血容量X」 |
| 2.2 | 从不同方向/高度受击 | 命中部位随命中点高度与左右变化（头/胸/腹/臂/腿） |
| 2.3 | 受较重切割/穿刺伤后静止观察 | 血容量持续缓慢下降（外出血），随后凝血使出血逐渐停止 |
| 2.4 | 持续失血直到血量很低 | 血量低于最大 15% 时死亡（见第 7 节） |
| 2.5 | `/livingsystem debug` | 查看各部位结构完整度与当前伤势列表 |

> 伤害画像数据：`fall`/`anvil`/`arrow`/`player`/`mob`（`data/livingsystem/damage_profile/`）。

## 3. 疼痛（阶段二）

| # | 操作 | 预期 |
| --- | --- | --- |
| 3.1 | 受多处创伤累积疼痛 | 总疼痛上升；移动/攻速轻微下降，镜头轻微摇晃 |
| 3.2 | 疼痛很高时 | 出现颤抖（手部稳定下降、摇晃加重） |
| 3.3 | 服用止痛药后（见第 5 节） | 镇痛使总疼痛下降，疼痛症状减轻 |
| 3.4 | 调 `painIntensityMultiplier` | 数值越大疼痛与惩罚越明显 |

## 4. 骨折与全身生理（阶段二）

### 4.1 骨折
| # | 操作 | 预期 |
| --- | --- | --- |
| 4.1.1 | 高处摔落 / 被坠落铁砧砸 | 较高概率出现「骨折 等级N (移位)」聊天提示 |
| 4.1.2 | 腿部骨折后移动 | 跛行：移动显著下降、无法疾跑，重度时跳跃受限 |
| 4.1.3 | 手臂骨折 | 攻速/挖掘/手部稳定下降 |
| 4.1.4 | 调 `fractureChanceMultiplier` | 调大更易骨折，便于测试 |

### 4.2 体力
| # | 操作 | 预期 |
| --- | --- | --- |
| 4.2.1 | 持续冲刺/跳跃/游泳 | 体力下降 |
| 4.2.2 | 静止休息 | 体力恢复（营养或水分耗尽时不恢复） |

### 4.3 呼吸 / 缺氧 / 意识
| # | 操作 | 预期 |
| --- | --- | --- |
| 4.3.1 | 长时间潜水（无水肺/水下呼吸） | 呼吸储备下降，耗尽后氧债累积 |
| 4.3.2 | 氧债累积到上限 | 缺氧终末死亡（见第 7 节） |
| 4.3.3 | 出水恢复 | 呼吸储备回升，氧债清偿 |

## 5. 治疗、药物与摄入（阶段三）

获取物品：创造模式物品栏搜索「livingsystem」，或用 `/give`。物品 ID：
`bandage`/`hemostatic_gauze`/`tourniquet`/`splint`/`suture_kit`/`antiseptic`/`painkiller`/`gas_mask`/`medical_supply`。

| # | 操作 | 预期 |
| --- | --- | --- |
| 5.1 | 出血时手持**绷带**右键自己并保持静止 | 出现治疗进度条；完成后外出血降低、凝血加快 |
| 5.2 | 治疗中移动 | 治疗中断（进度条消失） |
| 5.3 | 大出血时用**止血带**/**止血纱布** | 快速止血（止血带完成时几乎完全止血） |
| 5.4 | 骨折时用**夹板** | 完成后骨折不稳定下降、移位复位 |
| 5.5 | 开放伤口用**缝合包** | 伤口闭合，再出血风险大降，加速后续愈合 |
| 5.6 | 污染伤口用**消毒剂** | 污染程度按进度下降 |
| 5.7 | 服用**止痛药**（右键吃下） | 一段时间内镇痛，疼痛症状减轻 |
| 5.8 | 食用任意食物 | 补充代谢能量/营养/水分（原版饱食度被屏蔽） |

> 治疗会自动选择最相关、最严重的目标伤势；同一槽位不可重复发起。

## 6. 自然恢复（阶段三）

| # | 操作 | 预期 |
| --- | --- | --- |
| 6.1 | 控制出血+清洁后静养（保持营养/水分/休息） | 血容量缓慢回补；伤口逐步愈合、严重度下降 |
| 6.2 | 愈合完成 | 伤势消失（`/livingsystem debug` 不再列出）；结构完整度回升 |
| 6.3 | 缝合伤口 vs 未缝合 | 缝合显著加快愈合 |
| 6.4 | 愈合期间剧烈活动 | 愈合明显减缓 |
| 6.5 | 营养/水分耗尽 | 恢复能力大幅下降，几乎不愈合/不造血 |

## 7. 死亡与死亡报告（阶段二/三 3.6）

| # | 触发 | 预期死因摘要（聊天 §c[生命系统]） |
| --- | --- | --- |
| 7.1 | 失血至 <15% | 失血性休克 |
| 7.2 | 氧债满（淹溺） | 缺氧窒息 |
| 7.3 | 毒素/感染/辐射致死 | 对应死因（见第 9 节） |
| 7.4 | 意识极低 | 昏迷：移动/攻击被大幅锁定、禁疾跑跳 |

> 死亡报告同时通过事件分发给已订阅的兼容模组（见第 11 节）。

## 8. 医疗检查与健康界面（阶段三 3.5）

| # | 操作 | 预期 |
| --- | --- | --- |
| 8.1 | 按 **H** 打开健康界面 | 左侧七部位健康度（绿→黄→红 + 百分比），右侧体征摘要（血容量/体力/水分/呼吸/疼痛/意识），下方当前伤势列表（部位·类型·严重度，标注出血/骨折/已处理） |

## 9. 环境、防护、毒素、病原体、辐射（阶段四/五）

### 9.1 体温（环境）
| # | 操作 | 预期 |
| --- | --- | --- |
| 9.1.1 | 进入沙漠/下界等炎热环境 | 核心体温升高，>38.5℃ 出现发热（乏力） |
| 9.1.2 | 进入雪原/粉雪/冰冻 | 体温下降，<35.5℃ 出现寒战（减速、操作不稳、摇晃） |
| 9.1.3 | 寒冷环境穿戴**皮革护甲** | 隔热使体温下降减缓（`leather_insulation` 防护） |
| 9.1.4 | 接触岩浆/着火 | 环境温度骤升 |

### 9.2 防护
| # | 操作 | 预期 |
| --- | --- | --- |
| 9.2.1 | 穿戴护甲后受击 | 创伤严重度/结构损伤/骨折概率被防护减免 |
| 9.2.2 | 佩戴**防毒面具**进入毒气/烟雾 | 呼吸类暴露大幅降低（`gas_mask` 呼吸过滤 0.9） |

### 9.3 毒素 / 病原体 / 辐射
| # | 操作 | 预期 |
| --- | --- | --- |
| 9.3.1 | 站在**凋零玫瑰**上 | 吸入凋零毒素累积；恶心症状（移动下降/暗角/摇晃）；高剂量致死 |
| 9.3.2 | 站在**火/灵魂火**方块中 | 吸入烟雾，呼吸储备下降 |
| 9.3.3 | 让伤口高度污染并等待潜伏（默认 1200 刻≈60 秒） | 继发伤口感染：潜伏→症状期，出现发热/虚弱；免疫不足时载量升高，极高时致死 |
| 9.3.4 | 处于辐射环境（需自定义/数据包 RADIATION 危害） | 伽马剂量累积，超阈值出现恶心/虚弱，致死剂量致死 |

> 数据：`wither_rose_toxin`/`smoke_inhalation`/`nether_heat`（环境危害），`wither_toxin`/`wound_infection`/`gamma`（毒素/病原体/辐射定义）。

## 10. 持久化（伤势存档）

| # | 操作 | 预期 |
| --- | --- | --- |
| 10.1 | 造成若干创伤（含出血/骨折/已应用治疗），退出存档再进入 | 创伤及其组件（出血/骨折/疼痛/污染/异物/愈合/已应用治疗）、生理状态、结构完整度均保留 |
| 10.2 | 死亡后重生 | 伤势清空（默认不跨重生复制，重生为健康状态） |

> 注意：治疗会话、药物剂量、毒素/感染/辐射等其它健康影响目前为运行时态，重登录不保留（多为短时效数据）。

## 13. 已知限制 / 未实现（避免误判为缺陷）

- 医疗物品（绷带/夹板等）**无专用贴图**，显示为缺贴图紫黑（工作站复用铁块贴图正常）。属阶段七资源。
- 魔法异常（ARCANE）系统未实现（规格待细化）。
- 医疗**生产/治疗设备**（机器角色/配方/能源接入）、外部科技模组配方接入、完整管理界面未实现。
- 动态环境危害发射源、解毒/免疫增强药物未实现。
- 派生生命体征中除灌注/休克/总疼痛外多为占位值（医疗设备精确读数待阶段六设备）。
- 音效（心跳/呼吸/耳鸣）、画面 shader、治疗动画资源未实现。
- 完整昏迷输入锁定为近似（移动/攻击压制），非视角/挖掘全锁定。

## 14. 回归要点（每次改动后）

1. `./gradlew compileJava` 通过、无新增 deprecation。
2. `./gradlew runServer` 启动到 `Done` 无崩溃、无 `invalid dist`。
3. 数据包加载日志：伤害画像/治疗/环境危害/防护/毒素/病原体/辐射定义数量符合预期。
4. 专用服务器不加载任何 `net.minecraft.client.*`（dist 安全）。
